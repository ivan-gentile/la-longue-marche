## 1. **LLM-as-Judge Evaluation (NEW Section 4.8.2)**

**What was added:**
- Structured multidimensional evaluation rubric with 5 dimensions:
  - `text_accuracy` (1–5): Fidelity of French prose
  - `math_accuracy` (1–5): Correctness of mathematical symbols and structure
  - `completeness` (1–5): Coverage of all visible content
  - `formatting_quality` (1–5): LaTeX compilability and rendering
  - `overall` (1–5): Holistic utility assessment

**Why it matters:**
- String-level metrics (edit distance, sequence matching) are misaligned with actual evaluation goals
- Format variance (e.g., `$E$` vs. `$\mathcal{E}$`) and LaTeX synonyms penalize correct readings
- LLM-as-judge aligns evaluation with downstream use (human review, publication quality)

**Implementation:**
- Judge model: Gemini Flash-Lite (cost ~$0.001–$0.002 per judgment)
- Applied to stratified subsample (5 pages, ~25 judgments) for calibration
- Validation against expert human ratings required before scaling
- Results table (Table 1a) prepared with TBD placeholders

## 2. **Two-Pass Transcription Protocol (NEW Section 4.10)**

**What was added:**
- Architectural solution to prompt ambiguity and over-formatting failure mode
- **Pass 1 (Raw Reading):** Focus on content extraction with uncertainty markers
  - Explicit confidence annotation: `[HIGH]`, `[MED]`, `[LOW]`
  - Ambiguity flagging: `[?word?]` with alternatives
  - Non-formatted diagram/margin flagging: `[DIAGRAM: ...]`, `[MARGIN: ...]`
  - *No attempt at visual layout reproduction*

- **Pass 2 (Formatting):** Clean LaTeX generation from Pass 1 output
  - Operates on text only (no visual input)
  - Explicit layout suppression: "Do not add `\hfill`, `\vspace`, etc."
  - Resolves ambiguities and converts diagrams to LaTeX structures
  - Produces compilable, render-correct output

**Why it matters:**
- Single-pass "produce LaTeX" instructions are ambiguous between content extraction and visual reproduction
- Capable models interpret ambiguity by attempting layout reproduction → 39.5% `formatting_ratio` (wasted output capacity)
- Two-pass design prevents layout reproduction trap by decoupling reading from formatting

**Hypothesized outcome:**
- Lower `formatting_ratio` while maintaining or improving content accuracy
- Increased transparency into model uncertainty
- More efficient human-in-the-loop review (confidence-first)

**Testing:**
- Ablation study will compare single-pass, two-pass raw-reading, and hybrid approaches
- Primary evaluation metric: `formatting_ratio` reduction without content degradation

**Example output included:**
- Concrete examples of Pass 1 and Pass 2 output demonstrating the decoupling

**Cross-references:**
- Section 4.12 (Ablation Study): Tests two-pass as primary prompt style factor
- Section 6.5 (Discussion): Root cause hypothesis and model selection implications
- Section 7.1 (Future Work): Validation as immediate next step (Step 2)

## 3. **Input Format Ablation (Expanded Section 4.2, Phase C)**

- **Phase C: Input Format and Resolution Testing**
  - PNG-300dpi (current default): 2481×3192 px, 300 DPI render
  - PNG-150dpi (low resolution): 1240×1596 px, potential compression sensitivity test
  - PDF-direct: Native PDF page bytes, avoids rasterization artifacts entirely

**Research question:**
- Do VLMs internally normalize image resolution, making high-res gains negligible?
- Does PDF-direct avoid rasterization artifacts compared to PNG formats?

**Design:**
- 5 representative pages (diverse content types) × 3 formats × 2 models = 30 API calls
- Cost: ~$0.50

**Why it matters:**
- VLM vision encoders have fixed input grids; resolution may not translate to quality gains
- PDF-direct represents the "cleanest" input modality, avoiding any image compression

## 4. **Multi-Image Context (NEW Section 4.2, Phase D)**

**What was added:**
- **Phase D: Visual Context Testing**
  - Compare: No visual context (baseline) vs. Previous page as second image
  - Tests 8 consecutive page pairs where mathematical arguments span boundaries

**Key distinction from Phase A (text context):**
- Text-based context (Phase A) relies on extracted text, which loses notation and spatial layout
- Visual context preserves original visual appearance of notation and spatial relationships
- Hypothesis: Visual context better disambiguates notation (e.g., `\hat{X}` vs. `\tilde{X}`)

**Design:**
- 8 page pairs × 2 context modes × 1 best-performer model = 16 API calls
- Cost: ~$0.20

**Research question:**
- Does visual context preserve disambiguation information more effectively than text?
- Can previous page imagery help maintain notational consistency across page boundaries?

## 5. **Phased Experimental Design (Substantially Expanded Section 4.2)**

**What was added:**
- Complete restructuring from 2×2 factorial to phased sequential approach
- **Phase A (2×2 Factorial):** Segmentation × context
  - 80 API calls, ~$1.50
  - Research question: How do segmentation and text-based context affect quality?

- **Phase B (Model Comparison):** Best Phase A condition(s) across models
  - 60 API calls, ~$3.00
  - Research question: How do strategies generalize across VLM architectures?
  - Includes GPT-4o, Claude 3.5 Sonnet, open-source alternatives

- **Phase C (Input Format):** PNG resolution and PDF-direct
  - 30 API calls, ~$0.50

- **Phase D (Visual Context):** Previous page as second image
  - 16 API calls, ~$0.20

**Total cost:** ~186 API calls, ~$5.20 (**68% reduction** vs. full factorial design of ~280 calls)

**Cost-efficiency gains:**
- Early phases (A, B) answer high-priority questions with cheap models
- Phase B results guide model selection for C, D
- Later phases (C, D) can be skipped if earlier conclusions are sufficient
- Each phase provides interpretable insights rather than high-dimensional confounding

**Design benefits:**
- Sequential learning: Findings from early phases inform factor selection in later phases
- Cost-effective: Eliminates expensive multi-model × multi-condition crossings upfront
- Interpretable: Avoids factorial explosion and enables clear attribution of effects

**Cross-references:**
- Section 4.3 onwards: Models, prompts, evaluation metrics now reference phases
- Results section: Tables will be organized by phase
- Discussion: Findings contextualized within phased progression
- Future Work (Section 7.1): Detailed execution roadmap for all four phases

## 7. **Enhanced Discussion: Three-Tier Evaluation Methodology (NEW Section 6.1)**

**What was added:**
- Unified framework integrating all evaluation approaches:

  **Tier 1 — String-Similarity Metrics (SequenceMatcher):**
  - Quantitative, reproducible, but misaligned with evaluation goals
  - Use for: Ranking conditions, consistency checking
  - Avoid: Interpreting as absolute accuracy

  **Tier 2 — LLM-as-Judge Structured Evaluation:**
  - Multidimensional, aligned with use cases, but introduces judge bias
  - Use for: Validating string-metric rankings, interpretable feedback
  - Validate: Against expert human ratings

  **Tier 3 — Qualitative Expert Assessment:**
  - Ground truth but labor-intensive and non-reproducible
  - Use for: Catching phenomena missed by quantitative metrics
  - Integration: Informs interpretation of Tiers 1–2

**Advantages:**
- String metrics provide reproducible ranking without semantic insight
- LLM-as-judge validates that ranking aligns with human-relevant quality
- Qualitative assessment catches novel failure modes
- No single metric is trusted in isolation

**Cross-references:**
- Section 4.8 (Evaluation Metrics): All three tiers operationalized
- Section 5.0 (Results): LLM-as-judge subsample results
- Section 6 (Discussion): Framework interpretation of all results

## 8. **Expanded Over-Formatting Discussion with Two-Pass Solution (Revised Section 6.5)**

**Changes from original:**
- Deepened root-cause analysis:
  - Ambiguity in "produce LaTeX source code" (content extraction vs. visual reproduction)
  - Capable models interpret ambiguity by attempting both → over-formatting dominates

- Introduced two-pass protocol as explicit solution:
  - Pass 1: Focuses on reading (cannot reproduce layout)
  - Pass 2: Formats without visual input (cannot attempt layout reproduction)

- Added model selection implications:
  - Lower-capacity models paradoxically avoid over-formatting (cannot generate sophisticated layout)
  - Higher capacity can enable new failure modes if not carefully directed
  - Not an indictment of capability, but a call for better prompt design

- Linked to testing strategy:
  - Ablation study (Section 4.12) will directly test two-pass effectiveness
  - Predicted outcome: Lower `formatting_ratio`, maintained content accuracy

**Original content preserved:**
- `formatting_ratio` metric definition and empirical findings
- Implications for prompt design and model selection
- General principle: Capability ≠ always beneficial
